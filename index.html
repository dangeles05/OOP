Programación Orientada a Objetos (POO)

La programación orientada a objetos (POO) es un paradigma de programación que utiliza "objetos" para diseñar aplicaciones y programas. Los objetos pueden contener datos, en forma de atributos o propiedades, y código, en forma de métodos o funciones. Este enfoque permite que el software sea más modular, reutilizable y fácil de mantener.

Los Cuatro Pilares de la Programación Orientada a Objetos

1. Abstracción

La abstracción consiste en simplificar la complejidad del sistema al ocultar los detalles innecesarios y resaltar las características esenciales. En otras palabras, se trata de crear una representación simplificada de un objeto real, destacando solo lo que es relevante para el contexto del programa.

2. Encapsulamiento

El encapsulamiento se refiere a la agrupación de datos y métodos que operan sobre esos datos dentro de una misma unidad, o clase. Este concepto también implica la protección de los datos del acceso externo no autorizado, permitiendo el control sobre cómo se accede y modifica la información interna de un objeto.

3. Herencia

La herencia permite que una clase adquiera las propiedades y métodos de otra clase. Esto fomenta la reutilización del código y facilita la creación de jerarquías de clases, donde las clases hijas heredan el comportamiento y las características de las clases padre, con la posibilidad de añadir o modificar funcionalidades.

4. Polimorfismo

El polimorfismo permite que diferentes clases puedan ser tratadas de la misma manera, aunque implementen comportamientos diferentes. Esto se logra mediante el uso de interfaces o clases base que definen métodos comunes, permitiendo que las clases derivadas implementen esos métodos de manera específica.
